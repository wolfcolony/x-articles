OpenClaw新手完整学习路径

更适合新手食用的学习+使用教程

1个月以前，我也和你一样，看着 X 上铺天盖地关于OpenClaw的讨论，我心里痒痒的又不知道从哪儿下手。

那时候我最大的困惑是：

  • 网上教程太多了，我到底该按什么顺序学？

第一阶段：理解OpenClaw是什么（先看懂再动手）

阶段目标：建立对OpenClaw的基本认知，知道它能干什么、不能干什么，值不值得你花时间学。

很多新手上来就直接安装，结果发现根本不知道自己在干嘛。我的建议是，先花2-3小时看几个关键内容，建立心理预期。

必看资源

1. Nat Eliason的Felix案例

这是我见过最好的OpenClaw"可能性展示"。Nat Eliason用OpenClaw搭建了一个叫Felix的AI助手，三周内帮他赚了一万四千多美元。不是说你也能赚这么多钱，而是通过这个案例你能看到OpenClaw到底能做到什么程度。

  • 推荐观看：Peter Yang (@peteryang) 2月22日发的访谈视频
  • 配套阅读：creatoreconomy.so 上的详细文章《Use OpenClaw to Build a Business That Runs Itself》（付费文章，实际上可以只看视频即可）
  • 重点关注：Felix的3层记忆系统、多线程工作流、自主运营能力

2. 理解架构

OpenClaw不只像聊天机器人，它更像是一个能连上你系统和应用的"AI网关"。它能24/7运行、执行命令、管理文件，通过Telegram / WhatsApp / Discord 或者飞书来和你对话。

简单理解：ChatGPT活在浏览器里，OpenClaw活在你的电脑/服务器里。

如果看上面的视频你觉得复杂，你可以选择看 @GoSailGloba 老师的这篇解读，100万阅读的干货。

新手友好提醒

⚠️ 不要急着安装：看完这些内容后再决定要不要继续。OpenClaw需要你投入时间学习，也会产生API费用（高质量的API，比如Claude Opus 4.6，这个费用上限会很高，取决于使用频率）。

⚠️ 管理预期：OpenClaw很强大，但它不是魔法。前期需要大量配置和调教，以及逐渐去开放很多的授权，才能变成真正懂你的助手。

⚠️ 安全意识：因为OpenClaw能执行系统命令，配置不当可能会有安全风险。这也是为什么我强调要先理解原理。

第二阶段：基础搭建（让它跑起来）

阶段目标：完成最基础的安装和配置，让OpenClaw能响应你的消息。

这是很多人卡住的地方。我当时在Node.js版本上就折腾了半天。下面是我整理的最稳妥的安装流程。

1、前期准备（必须完成）

这里如果想要去看更详细的教程，或者遇到问题需要解决，我推荐范凯老师的教程，新手友好，我是参考这个来安装的。

系统要求

  • macOS / Linux / Windows（必须用WSL2）
  • Node.js v22或更高版本
  • 一个API密钥（Anthropic的Claude或OpenAI的GPT）
  • 一个Telegram账号（最简单的消息通道）
没有Telegram账号的，需要注册，这里需要海外手机号，可以去我主页里面找，有香港、荷兰、德国手机号的各种保姆级教程可以参考。
重要：安装过程记得打开科学上网！！！

Node.js安装

这是最容易出问题的环节。OpenClaw 2026版本严格要求Node.js v22+，用老版本会报错。如果你和我一样，使用的是很老版本的Mac，可以看我之前的这篇教程来升级Node版本。

推荐用nvm（Node Version Manager）来管理：

# 安装nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装Node.js v22
nvm install 22
nvm use 22

# 验证版本
node --version # 应该显示v22.x.x

2、一键安装脚本（官方最推荐方式）

# 官方安装脚本
curl -fsSL https://openclaw.ai/install.sh | bash

3、配置向导（安装完立刻运行）

openclaw onboard

这个向导会引导你完成：

  • 选择AI模型提供商（Anthropic / OpenAI / 本地模型）
  • 输入API Key
  • 选择消息平台（Telegram / WhatsApp / Discord）
  • 配置系统权限（建议先选沙盒模式）

API Key获取

  • Anthropic Claude：console.anthropic.com（推荐，$20/月 Pro 订阅）
  • OpenAI GPT：platform.openai.com
  • API聚合服务：https://aigocode.com/invite/MJA8A8MB（支持多模型，新用户有免费额度）

4、安装后必备命令（建议立刻保存）

# 查看配置

# 查看日志
openclaw logs

# 打开Dashboard（Web界面）

# 停止服务

5、想跳过向导快速测试？（很多人这么干）

# 跳过向导直接安装
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --no-onboard

openclaw dashboard # 直接打开浏览器测试

6、Telegram机器人设置

  1. 在Telegram中搜索 @BotFather
  2. 发送 /newbot 创建新机器人
  3. 设置名称和用户名
  4. 获取Bot Token（格式：123456789:ABCdefGHIjklMNOpqrsTUVwxyz）
  5. 配置到OpenClaw：
    openclaw config set channels.telegram.botToken "你的Token"
    openclaw config set channels.telegram.enabled true
  6. 在Telegram中搜索你的机器人并开始对话

7、验证安装

给你的机器人发送：

8、新手友好提醒

安装失败常见问题：

  • Node版本不对：必须v22+，用nvm切换
  • 权限不足：Mac需要给终端完全磁盘访问权限
  • 网络问题：确保开了科学上网
  • M1/M2 Mac：可能需要Rosetta 2

Mac用户如果遇到node-gyp错误，试试：

xcode-select --install
SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest

Linux用户如果遇到权限问题：

sudo apt install build-essential python3
npm install -g node-gyp
npm install -g openclaw@latest

第三阶段：核心配置（让它真正懂你）

阶段目标：通过配置文件让OpenClaw理解你的工作方式，建立长期记忆。

1、理解核心文件

OpenClaw的配置都在~/.openclaw/目录下，核心文件包括：

  • SOUL.md - 定义AI的身份、语气、行为准则
  • USER.md - 你的个人信息、偏好、工作习惯
  • MEMORY.md - 长期记忆存储
  • HEARTBEAT.md - 定时任务配置

2、SOUL.md配置最佳实践

SOUL.md决定了你的助手"是什么样的人"。

# SOUL: [给你的助手起个名字]

## 核心身份
- 你是一个高效的执行助手
- 你的语气简洁、技术化、直奔主题
- 你把每个任务当成使命，不达目标不罢休

## 行为准则
- 优先使用本地CLI工具，而不是网页界面
- 如果任务超过3步，先创建PLAN.md再开始执行

## 安全边界（重要！）
- **严格规定**：修改.env或credentials/文件夹前必须经过我的二次确认
- **隐私保护**：如果在日志中发现个人身份信息，发送给外部API前必须脱敏
- **财务限制**：单笔超过$50的操作必须在Telegram里获得我的"Y"确认

## 永远不要做的事
- 不要说"我很乐意帮助"、"好问题"这种客套话
- 不要在没有明确指令的情况下删除文件
- 不要在我睡觉时间（23:00-07:00）发送非紧急通知

3、USER.md配置技巧

告诉OpenClaw关于你的一切：

  • 你的名字、职业、时区
  • 工作时间、回复习惯
  • 常用的项目、工具
  • 个人偏好（如"我喜欢简洁的回复"）

4、3层记忆系统（Nat Eliason的方法）

  1. 短期记忆：当前对话上下文
  2. 中期记忆：session间的项目信息
  3. 长期记忆：你的偏好、习惯、重要事实

5、新手友好提醒

SOUL.md不是一次性写好的，建议先用一个月，边用边改。观察助手的回复，不满意的地方就加到SOUL.md里。

可以参考这些优秀案例：

  • @lijiuer92 的SOUL.md模板
  • Matthew Berman的自动化配置
  • Jacky Chou的商业化案例

第五阶段：进阶优化（效率最大化）

阶段目标：让OpenClaw成为真正的"第二大脑"。

1、Model Routing（省钱的关键）

不同任务用不同模型，大幅降低API成本：

  • 简单任务：用GPT-3.5或Claude Haiku（便宜）
  • 中等任务：用Claude Sonnet（平衡）
  • 复杂任务：用Claude Opus（贵但强）

参考 @justinweb33 的Model Routing配置。

2、24/7 运行（VPS 部署）

把OpenClaw部署到云服务器，实现24小时在线：

  • 推荐平台：DigitalOcean、AWS、阿里云
  • 最低配置：1核2G内存
  • 成本：约$5-10/月

3、Cron 任务（自动化）

设置定时任务，让OpenClaw自动执行：

  • 每天早上8点发天气+日程简报
  • 每周五下午生成本周总结
  • 每天自动整理下载文件夹

参考 JaredL 的自动化案例。

4、多 Agent 协作（高级玩法）

运行多个OpenClaw实例，每个负责不同任务：

  • 工作Agent：处理邮件、会议、项目
  • 个人Agent：管理日程、提醒、笔记
  • 研究Agent：专门做信息收集和分析

5、API 成本监控工具

推荐：https://clawwatcher.com

功能：实时监控API调用次数和费用，设置预算上限，防止意外超支。

6、新手友好提醒

进阶优化不要着急，建议先用熟基础功能，至少1-2个月后再考虑。先把一个Agent调教好，比同时运行3个半吊子Agent有用得多。
